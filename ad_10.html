Тестування готового сайту

Тестування сайту крос браузерно та за допомгою валідатору

Перевіримо наш шаблон на валідність. Заходимо на сайт http://validator.w3.org/#validate_by_upload і вибираємо файл index.html . Якщо ви все зробили так само, як описано в уроках, то помилок в документі бути не повинно, і у вас на екрані з'явиться таке повідомлення

Тепер вам потрібно завантажити і встановити такі браузери, щоб протестувати сторінку в них:

1.Mozilla Firefox

2.Opera

3.Safari (только для iOS и MacOS)

4. Yandex

5. Программу IE Tester для тестування веб-сторінки в різних версіях браузера Internet Explorer.

Якщо виникають проблеми з кодуванням як в відео, то ось файл htaccess , який потрібно вставити, щоб виправити це в IE7.

Відкриваємо шаблон index.html в браузері Mozilla Firefox . Шрифт на кнопках не відповідає тому, який ми встановили:

Тому відкриваємо файл style.css і для селекторів input,button,textarea (приблизно 44 рядок у файлі) дописуємо рядок font-family: inherit; . Значення inherit означає, що дана властивість успадкує значення батька, в нашому випадку селектора body :

                input,button,textarea {
                box-sizing: border-box;
                -webkit-box-sizing: border-box;
                -moz-box-sizing: border-box;
                display: inline-block;
                border-radius: 3px;
                -webkit-border-radius: 3px;
                -moz-border-radius: 3px;
                font-family: inherit;
                }

Після оновлення сторінки ми побачимо, що заробив потрібний нам шрифт.

Небольшое дополнение про шрифты. Вы можете установить несколько названий шрифтов для того, чтобы исключить вероятность отсутствия определенного шрифта на компьютере пользователя. Если название шрифта имеет пробелы, то оно должно записываться в кавычках. Закончить список названий можно ключевым словом, которое будет описывать тип шрифта:

  1. serif – шрифты с засечками, например, Georgia, Times New Roman, Times;
  2. sans-serif – шрифты без засечек, например, Verdana, Geneva, Tahoma;
  3. cursive – курсивные шрифты;
  4. fantasy – декоративные шрифты;
  5. monospace – моноширинные шрифты, например, Courier.

До властивості font селектора body допишемо додаткові шрифти, і в кінці вкажемо ключове слово sans-serif , яке описує тип перерахованих шрифтів:

                        body {
                        background: #fbf7f5 url("img/bg.jpg") repeat;
                        font: 14px/22px Arial, Tahoma, Verdana, sans-serif;
                        color: #444;
                        }

Тепер відкриваємо інші браузери: Opera , Safari , Yandex , і перевіряємо веб-сторінку в них. У мене все відображається коректно, тому переходимо до самого проблемного браузеру - це браузер Internet Explorer (IE) .

Запустіть програму IE Tester , і відкрийте файл index.html у вкладках для 7, 8 і 9 версії:

У будь-якій версії браузера IE зображення, які є посиланнями, виділені блакитний кордоном. Щоб цю межу прибрати, потрібно в файлі зі стилями записати наступні рядки:

                    a img {
                        outline: 0;
                        border: 0;
                    }

Властивість outlin eвідповідає за зовнішню кордон.

У сьомий і восьмий версіях браузера IE текст, що вводиться в текстовому полі , і кнопка Знайти прибиті до верху:

Також не відображається тінь у блоків і кнопок, так як дані версії не розуміють властивість box-shadow .

Для текстового поля ми поставимо міжрядковий інтервал з таким значенням: line-height: 2; , а для блоків пропишемо кордон, щоб вони стали більш помітні. Як зробити так, щоб цей властивості працювали тільки для браузера IE 7 і 8 версії?

Для цього ми створимо окремий файл, назвемо його ie.css . У ньому пропишемо властивості:

                    input[type="text"] {
                        line-height: 2;
                    }
                    
                    .block>.content,
                    .block.category .v-menu li a {
                        border: 1px solid #ccc;
                        border-top: 0;
                    }

І підключимо цей файл за допомогою умовного коментаря в файлі index.html в контейнері <head>:

                    <!--[if (ie 7) | (ie 8)]><link rel="stylesheet" href="ie.css"><![endif]-->

В даному випадку код буде працювати для сьомий чи восьмий версії браузера IE .

З восьмої версією розібралися, переходимо до сьомої. Тут не працює властивість box-sizing , тому текстове поле в блоці для пошуку товарів за межі блоку:

Також сьома версія не розуміє значення inline-block властивості display , тому відображає посторінкову навігацію як блоковий елемент, через що вона розтягується на всю ширину контентної частини:

Ми запишемо властивості в файлі ie.css, які будуть працювати тільки для сьомої версії браузера IE, для цього перед цими властивостями потрібно поставити зірочку, інакше кажучи скористаємося Хаком :

                    .block.search form>input[type="text"] {
                        *width: 137px;
                    }

Тут ми вказали ширину текстового поля без урахування внутрішніх відступів і кордонів. З урахуванням цих параметрів ширина повинна складати 200px .

200px-(1px+1px+4px+57px)=137px

Значення в дужках - це сума з ширини лівої межі, правої межі, лівого внутрішнього відступу і правого внутрішнього відступу.

Хак - це особливе написання селектор або властивостей CSS для того, щоб вони були зрозумілі лише певному браузеру.

Опустимо трохи кнопку до центру текстового поля:

                    .block.search form>button {
                        *top: 7px;
                    }

Для посторінковою навігації запишемо наступні властивості:

                    .pagination {
                        *margin-top: 20px;
                        *display: inline;
                    }

Тепер наш шаблон в ранніх версіях браузера IE відображається коректно, і етап тестування завершено.
